WHAT HAPPENS WHEN YOU DRAG AN ISSUE
================================================================================

1. USER ACTION:
   - User drags issue card from "To Do" to "In Progress" column

2. JAVASCRIPT:
   - HTML5 Drag API fires "drop" event
   - JavaScript gets issue key (e.g., "BP-1") and target status ID (e.g., 2)
   - Sends POST request to: /api/v1/issues/BP-1/transitions
   - Body: {"status_id": 2}

3. API ENDPOINT (IssueApiController::transition):
   - Validates status_id is an integer
   - Validates issue exists
   - Calls IssueService::transitionIssue()

4. SERVICE LAYER (IssueService::transitionIssue):
   - Checks if transition is allowed (workflow rules)
   - Checks target status exists
   - Calls Database::update()
   - Records history entry
   - Logs audit trail
   - Returns updated issue

5. DATABASE LAYER:
   - Executes: UPDATE issues SET status_id = 2 WHERE id = 1
   - Uses PDO prepared statement (no SQL injection possible)
   - Changes committed to database immediately
   - Transaction atomic (succeeds or fails completely)

6. RESPONSE:
   - Server responds: {"success": true, "issue": {...}}
   - Returns fresh data from database (not cached)

7. JAVASCRIPT UI UPDATE:
   - Card moves in UI
   - Column counts update
   - Shows success message

8. PERSISTENCE VERIFICATION:
   - User reloads page (F5)
   - Board re-renders from database
   - Issue appears in NEW column
   - Database query confirms status_id = 2
   - ✅ DATABASE CHANGE CONFIRMED
