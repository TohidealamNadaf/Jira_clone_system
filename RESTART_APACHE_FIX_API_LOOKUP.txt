HOW TO RESTART APACHE
─────────────────────

OPTION 1: Using Laragon (Easiest)
──────────────────────────────────
1. Look for Laragon icon in system tray (bottom-right Windows)
2. Click Laragon icon → Opens Laragon dashboard
3. Find "Apache" button with status indicator
4. Click "Restart" button (or stop then start)
5. Wait for "Apache started" message

OPTION 2: Using Laragon Menu
──────────────────────────────
1. Click Laragon icon
2. Find "Tools" or "Services"
3. Click "Apache" → "Restart"
4. Wait for confirmation

OPTION 3: Using XAMPP Control Panel
────────────────────────────────────
1. Find XAMPP Control Panel in Windows Start
2. Click "Apache" → "Stop"
3. Wait 2-3 seconds
4. Click "Apache" → "Start"
5. Verify status shows "Running"

OPTION 4: Command Line (Advanced)
──────────────────────────────────
1. Open Command Prompt (Win+R, type "cmd", press Enter)
2. Type: net stop Apache2.4
3. Wait for confirmation
4. Type: net start Apache2.4
5. Wait for "Service started successfully"

AFTER RESTARTING
────────────────
1. Clear browser cache
   - Press CTRL+SHIFT+DEL
   - Select "Cached images and files"
   - Click "Clear"

2. Hard refresh browser
   - Press CTRL+F5 (or CTRL+Shift+R on Mac)
   - This forces browser to reload JavaScript

3. Test the fix
   - Open Create Issue Modal
   - Check console (F12) for error messages
   - Should see: ✅ Issue types loaded
   - Should see: ✅ Loaded priorities

4. Verify dropdowns work
   - Click Issue Type dropdown → should show options
   - Click Priority dropdown → should show options
   - No more "404 Not Found" errors
