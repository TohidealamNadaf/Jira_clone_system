### Step 4: Key Concepts (5 minutes)

**Projects:** Containers organizing all work
- Example: "Mobile App", "Backend API", "Frontend"
- Each has issues, sprints, and team members

**Issues:** Individual work items (bugs, features, tasks)
- Assigned to people
- Move through workflow (To Do ‚Üí Done)
- Team collaborates via comments

**Sprints:** Time-based iterations (usually 2 weeks)
- Contains set of issues to complete
- Has deadlines and goals
- Generates metrics and reports

**Board:** Visual Kanban view of workflow
- Columns: To Do, In Progress, In Review, Done
- Drag issues to change status
- Real-time progress view

**Status:** Where issue is in workflow
- **To Do:** Not started
- **In Progress:** Being worked on
- **In Review:** Waiting for approval
- **Done:** Completed

## üìù Day 3: First Tasks & Hands-On Practice

### For All Roles: Complete These Tasks

#### Task 1: Explore a Project (20 min)
1. Go to Projects
2. Pick one project
3. Click to view it
4. Look at:
   - Project overview
   - Team members
   - Recent issues
   - Board view
   - Sprint schedule

#### Task 2: Understand an Issue (15 min)
1. Find an issue on board
2. Click it to open
3. Read:
   - Title and description
   - Status and assignee
   - Comments from team
   - Any attachments
   - Related issues

#### Task 3: Perform Your Role Action (20 min)

**If Developer:**
- Find unassigned issue
- Assign to yourself
- Click to "In Progress"
- Add comment with update
- Get feedback

**If Project Manager:**
- View sprint board
- Check how many issues done
- Check burndown trend
- Identify any stuck issues
- Make note for standup

**If QA/Tester:**
- Find a closed issue
- Re-test the original problem
- Create new issue if bug still exists
- Or confirm "verified fixed"
- Add comment with test results

**If Viewer/Stakeholder:**
- View sprint board
- Count issues in each column
- Check burndown chart
- Estimate progress
- Write down questions

#### Task 4: Add a Comment (10 min)
1. Pick any open issue
2. Scroll to Comments section
3. Type a comment (can be test comment)
4. Mention someone: `@username`
5. Click Send
6. See it appear instantly

#### Task 5: Get Feedback (15 min)
1. Show your team lead what you did
2. Ask if you did it correctly
3. Get tips for doing it better
4. Ask any questions that came up
