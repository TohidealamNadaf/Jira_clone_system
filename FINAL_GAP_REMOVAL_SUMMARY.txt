PROBLEM: Visible orange-highlighted gaps on every page (top, left, right)

ROOT CAUSE #1 (PRIMARY): Inline Style with Highest Specificity
───────────────────────────────────────────────────────────────
File: views/layouts/app.php, Line 1147

BROKEN CODE:
    <div style="max-width: 1400px; margin: 0 auto; padding: 32px; width: 100%;">
                                                    ^^^^^^^^^^^^^^^^^^^^^^
                                                    32px padding causing gaps

FIXED CODE:
    <div style="width: 100%;">
    (removed all padding - let page wrappers handle it)

WHY THIS MATTERS:
- Inline styles have HIGHEST CSS specificity (except !important)
- Even if CSS is correct, inline styles ALWAYS win
- This was the PRIMARY cause of the gaps on every page


ROOT CAUSE #2 (SECONDARY): Conflicting CSS Rule
─────────────────────────────────────────────────
File: public/assets/css/design-consistency.css, Lines 18-23

BROKEN CODE:
    #mainContent > div {
        padding: 32px;  ← Adding 32px to all divs inside main
    }

FIXED CODE:
    /* REMOVED */

WHY THIS MATTERS:
- Was adding secondary padding even if inline style fixed
- Too broad selector targeting all direct children
- Conflicted with page wrapper padding from app.css


═════════════════════════════════════════════════════════════════════════════════
                           HOW THE FIX WORKS NOW
═════════════════════════════════════════════════════════════════════════════════

FLOW:
1. User navigates to page (e.g., /search)
2. View file renders with wrapper: <div class="search-page-wrapper">
3. CSS applies: .search-page-wrapper { padding: 1.5rem 2rem; }
4. Result: 24px top/bottom padding + 32px left/right padding
5. Seamless navbar-to-content transition, NO gaps
