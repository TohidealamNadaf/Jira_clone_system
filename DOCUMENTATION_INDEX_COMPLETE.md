## Documentation Standards

### File Naming
- **Feature docs**: `FEATURE_NAME.md`
- **Fix docs**: `FIX_ISSUE_DESCRIPTION.md`
- **Status docs**: `*_SUMMARY.md` or `*_STATUS.md`
- **Quick ref**: `*_QUICK_*.md`
- **Implementation**: `*_IMPLEMENTATION.md`

### File Structure
```
# Title
Description
## Section 1
Content
## Section 2
Content
```

### Content Standards
- Use markdown formatting
- Include code examples
- Provide step-by-step instructions
- Add visual guides when helpful
- Link to related documentation
